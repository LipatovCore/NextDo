# Архитектура

## Компоненты

- `config` — Django-проект: настройки, корневые маршруты, WSGI/ASGI.
- `task` — Django-приложение задач.
- `templates` — серверные HTML-шаблоны с CSS, partial-шаблонами задач и небольшим JavaScript в рабочих страницах задач и проектов.
- `db` — PostgreSQL 15 в Docker Compose.
- `web` — Django + Gunicorn.
- `nginx` — reverse proxy на `web:8000` и отдача `/static/`.

## Модель данных

`Task` хранит:

- `title` — `CharField(max_length=100)`.
- `user` — внешний ключ на `settings.AUTH_USER_MODEL`.
- `project` — необязательный внешний ключ на `Project`; при удалении проекта очищается через `SET_NULL`.
- `is_completed` — флаг завершения, по умолчанию `False`.
- `priority` — приоритет `low`, `medium` или `high`; по умолчанию `medium`.
- `deadline` — дедлайн задачи, может быть пустым.
- `scheduled_date` — дата выполнения/планирования, может быть пустой.
- `description` — описание задачи.
- `is_deleted` — флаг мягкого удаления, по умолчанию `False`.
- `created_at` — дата создания.

`Project` хранит:

- `title` — `CharField(max_length=100)`.
- `user` — внешний ключ на `settings.AUTH_USER_MODEL`.
- `deadline` — обязательный дедлайн проекта.
- `created_at` — дата создания.

Сортировка по умолчанию: новые задачи и проекты сверху через `ordering = ['-created_at']`.

Обычные списки задач строятся только по `user=request.user` и `is_deleted=False`. Вкладка «Сегодня» показывает задачи с `scheduled_date` на текущий день, а также задачи, у которых `deadline` или `scheduled_date` строго раньше текущей даты. Страница проекта дополнительно фильтрует задачи по выбранному проекту; статистика проекта считает только неудалённые связанные задачи.

## Маршруты

- `/admin/` — Django admin.
- `/login/` — стандартный `LoginView` с шаблоном `registration/login.html`.
- `/logout/` — стандартный `LogoutView` с шаблоном `registration/logout.html`.
- `/` — отдельная главная страница.
- `/tasks/` — список и создание задач.
- `/projects/` — список и создание проектов.
- `/projects/<project_id>/` — редактирование проекта и список его задач.
- `/finance/` — отдельный защищённый раздел финансов.
- `/tasks/<task_id>/toggle/` — переключение статуса задачи.
- `/tasks/<task_id>/status/` — установка статуса из мини-карточки.
- `/tasks/<task_id>/today/` — добавление задачи в список на сегодня или очистка даты выполнения.
- `/tasks/<task_id>/detail/` — HTML полной карточки задачи.
- `/tasks/<task_id>/edit/` — сохранение полной карточки задачи.
- `/tasks/<task_id>/delete/` — мягкое удаление задачи через `is_deleted=True`.

## Поток запроса

1. nginx принимает HTTP-запрос.
2. nginx проксирует приложение на `web:8000`.
3. Gunicorn передает запрос в `config.wsgi`.
4. Django выбирает маршрут в `config.urls` или `task.urls`.
5. `task.views` работает только с задачами и проектами текущего пользователя.

Страница `/` рендерится отдельным серверным Django-шаблоном главной страницы. Страница `/tasks/` рендерит рабочий раздел задач. Страницы `/projects/` и `/projects/<project_id>/` используют тот же визуальный паттерн; действия с задачами внутри проекта передают контекст проекта в AJAX-запросах и получают обновлённый HTML списка вместе со статистикой проекта. Без fetch-запроса views продолжают работать через обычный render/redirect.

## Ограничения

- Регистрация пользователей в коде не найдена; пользователей можно создавать через admin, management command или shell Django.
- Отдельный REST API не выделен; JSON-ответы используются только для улучшения серверного интерфейса `/tasks/` и задач внутри `/projects/<project_id>/`.
- Фоновые задачи и внешние интеграции не найдены.
