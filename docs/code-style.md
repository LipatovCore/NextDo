# Стиль кода

## Найдено

- Проект использует обычную структуру Django.
- Python-код разделен по стандартным файлам приложения: `models.py`, `views.py`, `forms.py`, `urls.py`, `tests.py`.
- Шаблоны используют Django Template Language.
- CSS хранится внутри HTML-шаблонов в блоках `extra_style`.
- Страница задач и страница входа используют светлую рабочую тему с белыми панелями, синими основными кнопками и серыми границами.
- JavaScript для фильтра завершенных задач находится в `src/templates/task/task-list.html`.

## Не найдено

- Конфиг линтера.
- Конфиг форматтера.
- Конфиг типизации.
- CI-конфиг.

## Правила для изменений

- Сохраняйте существующий Django-подход: тонкие views, формы через `forms.ModelForm`, маршруты в `urls.py`.
- Не добавляйте зависимости без необходимости.
- Не переносите inline CSS/JS в отдельные файлы без явного запроса: текущая структура держит стили рядом с шаблонами.
- При добавлении полей модели создавайте миграцию и тесты.
- Не меняйте user isolation: views должны работать только с задачами `request.user`.
