# Обзор проекта

NextDo — Django-приложение для личного списка задач с рабочей страницей управления задачами.

## Возможности

- Вход и выход через стандартную аутентификацию Django.
- Просмотр неудалённых задач текущего пользователя.
- Создание задачи с названием до 100 символов.
- Переключение статуса задачи: активная или завершенная.
- Приоритет, дедлайн, дата выполнения и описание задачи.
- Вкладка «Сегодня» для задач на текущий день и просрочек.
- Фильтры по статусу, приоритету, дедлайну и дате выполнения.
- Полная карточка задачи в модальном окне.
- Мягкое удаление задачи через скрытие из обычных списков.
- Отдельная главная страница `/`.
- Отдельный защищённый раздел `/tasks/` для задач.
- Отдельный защищённый раздел `/finance/` для финансовой страницы-заглушки.

## Стек

- Python 3.13 в Docker-образе.
- Django 6.0.3.
- PostgreSQL 15 в Docker Compose.
- SQLite для тестов.
- Gunicorn для запуска Django в контейнере.
- nginx для входящего HTTP и статических файлов.

## Ключевые файлы

- `src/config/settings.py` — настройки Django, БД, статика, auth redirects.
- `src/config/urls.py` — маршруты admin, login, logout, home, tasks и finance.
- `src/task/models.py` — модель `Task`.
- `src/task/views.py` — обработчики списка, фильтров, карточки, статуса, даты выполнения и мягкого удаления задач.
- `src/task/tests.py` — тесты модели и views.
- `docker-compose.yml` — запуск PostgreSQL, web и nginx.
- `.env.example` — пример окружения.
