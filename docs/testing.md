# Тестирование

## Команда

```bash
python src/manage.py test task
```

## Что покрыто

`src/task/tests.py` проверяет:

- создание задачи;
- связь задачи с пользователем;
- переключение `is_completed`;
- значения по умолчанию для приоритета, дат, описания и мягкого удаления;
- признаки просрочки по дедлайну и дате выполнения;
- каскадное удаление задач при удалении пользователя;
- максимальную длину `title`;
- сортировку новых задач сверху;
- редирект анонимного пользователя на login;
- отображение только неудалённых задач текущего пользователя;
- создание задачи через POST с обрезкой лишних пробелов;
- обработку невалидной формы;
- главное меню и вкладки задач;
- фильтры по статусу, приоритету, дедлайну и дате выполнения;
- комбинирование фильтров;
- вкладку «Сегодня» для сегодняшних и просроченных задач;
- AJAX-создание, обновление статуса, управление датой «Сегодня», загрузку и сохранение карточки;
- изоляцию новых endpoint'ов от чужих задач;
- мягкое удаление задач через `is_deleted=True`.

## База данных тестов

В `src/config/settings.py` есть проверка `RUNNING_TESTS = 'test' in sys.argv`. При тестах Django использует SQLite:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'test_db.sqlite3',
    }
}
```

## Перед изменениями

- Добавляйте или обновляйте тесты при изменении модели, views, URL или поведения шаблонов.
- Если меняется авторизация или изоляция пользовательских задач, проверьте сценарии с чужими задачами.
